## Purpose
Provide a structured onboarding checklist for new team members joining OctoAcme projects, ensuring they understand their role, responsibilities, and how to collaborate effectively with other roles.

## When to use
Use this template when onboarding new team members to a project or when someone takes on a new role.

---

## General Onboarding (All Roles)

### Week 1: Project Context
- [ ] Read [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Review current project charter or one-pager
- [ ] Understand project goals and success metrics
- [ ] Meet the team and understand reporting structure
- [ ] Access project communication channels (Slack, Teams, etc.)
- [ ] Review [Roles & Personas](octoacme-roles-and-personas.md) document

### Week 1: Tools & Access
- [ ] GitHub repository access granted
- [ ] Project board access (GitHub Projects, Jira, etc.)
- [ ] Development environment setup (if applicable)
- [ ] Documentation repository access
- [ ] Calendar invites for recurring meetings

### Week 1: Process Familiarization
- [ ] Attend first standup or team sync
- [ ] Review current sprint/iteration plan
- [ ] Understand escalation paths and communication norms
- [ ] Shadow a team member in your role (if available)

## Role-Specific Onboarding

### Developers

#### First Week
- [ ] Review [Execution & Tracking](octoacme-execution-and-tracking.md) guide
- [ ] Set up local development environment
- [ ] Run existing tests and builds successfully
- [ ] Review coding standards and PR workflow
- [ ] Pair with another developer on a small task
- [ ] Meet with Technical Lead for architecture overview

#### First Month
- [ ] Complete first PR with code review
- [ ] Participate in design discussion or technical review
- [ ] Understand CI/CD pipeline and deployment process
- [ ] Shadow QA handoff process

---

### Product Managers

#### First Week
- [ ] Review product roadmap and current priorities
- [ ] Meet with stakeholders to understand business goals
- [ ] Review user research and customer feedback sources
- [ ] Understand success metrics and how they're measured
- [ ] Meet with Data Analyst to review analytics setup

#### First Month
- [ ] Lead or co-lead a sprint planning session
- [ ] Present roadmap update to stakeholders
- [ ] Conduct user research or customer interview
- [ ] Review and prioritize backlog items

---

### Project Managers

#### First Week
- [ ] Review [Project Initiation Guide](octoacme-project-initiation.md)
- [ ] Understand current project timeline and milestones
- [ ] Review risk register and active issues
- [ ] Meet with Product Manager and Technical Lead
- [ ] Understand team velocity and capacity

#### First Month
- [ ] Facilitate a team meeting (standup, planning, or sync)
- [ ] Update project status report and share with stakeholders
- [ ] Identify and mitigate a project risk
- [ ] Conduct stakeholder communication

---

### UX Designers

#### First Week
- [ ] Review product vision and user personas
- [ ] Access design tools and asset repositories
- [ ] Review existing design system and UI patterns
- [ ] Meet with Product Manager to understand feature priorities
- [ ] Review user research findings and usability reports

#### First Month
- [ ] Complete a design deliverable (mockup, prototype, etc.)
- [ ] Conduct a design handoff with developers
- [ ] Participate in user research or usability testing
- [ ] Present design in a stakeholder review

---

### Technical Leads

#### First Week
- [ ] Review technical architecture and design docs
- [ ] Understand technology stack and infrastructure
- [ ] Review code standards and best practices
- [ ] Meet with development team to understand current challenges
- [ ] Identify technical debt and risks

#### First Month
- [ ] Conduct code reviews and provide mentoring
- [ ] Make or document a technical decision (ADR)
- [ ] Lead a technical design discussion
- [ ] Partner with Project Manager on technical risk mitigation

---

### QA/Testing

#### First Week
- [ ] Review test plans and testing strategy
- [ ] Access test environments and tooling
- [ ] Understand release process and quality gates
- [ ] Meet with Developers to understand testability approach
- [ ] Review bug tracking and triage process

#### First Month
- [ ] Execute test plan for a feature
- [ ] Report and verify bug fixes
- [ ] Participate in release readiness review
- [ ] Provide feedback on testability during planning

---

### Data Analysts

#### First Week
- [ ] Review success metrics and KPIs
- [ ] Access analytics platforms and data sources
- [ ] Understand data instrumentation and tracking
- [ ] Meet with Product Manager to review metrics goals
- [ ] Review existing dashboards and reports

#### First Month
- [ ] Create or update a dashboard or report
- [ ] Define metrics for a new feature with Product Manager
- [ ] Review data instrumentation with Developers
- [ ] Present insights to stakeholders

---

### Support/Customer Success

#### First Week
- [ ] Review customer feedback channels and tools
- [ ] Understand product features and common use cases
- [ ] Access support documentation and knowledge base
- [ ] Meet with QA to understand bug triage process
- [ ] Shadow experienced support team member

#### First Month
- [ ] Handle customer issues and escalations
- [ ] Participate in bug triage meeting
- [ ] Provide customer feedback in retrospective
- [ ] Identify documentation gaps and suggest improvements

---

## Onboarding Checklist for Managers

When onboarding a new team member, the hiring manager or project lead should:

- [ ] Assign an onboarding buddy or mentor
- [ ] Schedule 1:1 meetings with key collaborators
- [ ] Provide access to all necessary tools and systems
- [ ] Share role-specific documentation and resources
- [ ] Set clear expectations for first week, month, and quarter
- [ ] Schedule regular check-ins (daily for first week, weekly for first month)
- [ ] Gather feedback on onboarding experience after first month

---

## Continuous Learning

All team members should:
- [ ] Participate in retrospectives and share learnings
- [ ] Stay current with process documentation updates
- [ ] Provide feedback on gaps or improvements to processes
- [ ] Mentor new team members when possible
- [ ] Review [Retrospective & Continuous Improvement](octoacme-retrospective-and-continuous-improvement.md) guide
